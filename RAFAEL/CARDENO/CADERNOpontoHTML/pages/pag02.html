CAP 25 Media Queryes

Simplismente, adaptações para diferentes situações principalemente com a mudança de tela do usuario, pois temos Tablets, Computadores, notebooks, celulares como tamanhos referentes onde o site tem de ser resposivo.
Podemos retirar, movimentar, alterar ou até adicionar os elementos que nao se encaixam com a mudança de tela

Syntax

<style>
@media not|only mediatype and (expressions) {
CSS-Code;
}
<style>

Faça em arquivos separados para MQ em sites mais extensos !!

Utilizando <link rel="stylesheet" href="estilos/tela ou impressora.css" media="screen ou print ou all"> podemos dividir e organizar nossas pastas media queryes, onde com o atributo media indicamos formatações exclusivas para screen (para telas) print (impressoras) all (padrao, para ambas)

Media features e media types

È como chamamos as medias features (screen print and all) e media types (orientation: landscape ou portrait) representando a media type desta forma media= "screen and (orientation: portrait ou landscape)"

Mobile first

Uma regra muito importante ao criarmos nossas MQ, é a vantagem de se começar primeiro pelo tamanho mobile atualmente em (320px — 480px.). Alem de garantir maior acessibilidade ao seu site, pois começar com a menor resolução só teremos que tirar ou adicionar elementos que nao se encaixam ao tamanho.
Vantagens - Facilidade na divulgação do site, de acordo com o criador desta regra o google foca as pesquisas em sites Mobile First. Melhoria na experiencia do usuario - Focar em Mobile first, faz com que foquemos em sites mais leves, o que favorece o acesso em locais com internet mais lenta. Alem de favorecer tambem na pesquisa do google novamente.
